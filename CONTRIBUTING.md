# 贡献指南

感谢您考虑为 LLM Gateway 项目做出贡献！我们欢迎各种形式的贡献，包括但不限于代码贡献、文档改进、问题报告和功能建议。

## 贡献方式

### 报告问题

如果您发现了bug或有功能建议，请先检查是否已有相关Issue。如果没有，请创建一个新的Issue，包含以下信息：

- **问题描述**：清晰描述问题或建议
- **重现步骤**：如何重现问题
- **环境信息**：操作系统、Python版本、依赖版本等
- **期望行为**：您期望看到的结果
- **实际行为**：实际看到的结果

### 代码贡献

1. **Fork 仓库**：点击GitHub页面右上角的Fork按钮
2. **创建分支**：为您的功能或修复创建新分支
3. **编写代码**：遵循项目的代码风格
4. **测试**：确保您的代码通过现有测试
5. **提交PR**：创建Pull Request并描述您的更改

### 代码风格

- 使用 **4个空格** 进行缩进
- 遵循 **PEP 8** Python代码风格指南
- 为函数和类添加适当的文档字符串
- 为复杂逻辑添加注释

### 提交信息规范

提交信息应遵循以下格式：

```
类型(范围): 简短描述

详细描述（可选）

关闭 #Issue编号（可选）
```

类型包括：
- `feat`: 新功能
- `fix`: bug修复
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建过程或辅助工具变动

示例：
```
feat(api): 添加新的API提供商支持

- 支持新的API提供商配置
- 添加相应的测试用例

关闭 #123
```

## 开发环境设置

1. **克隆仓库**
```bash
git clone https://github.com/your-username/llm-gateway.git
cd llm-gateway
```

2. **安装依赖**
```bash
pip install -r requirements.txt
```

3. **运行测试**
```bash
pytest
```

4. **启动开发服务器**
```bash
python app.py
```

## 测试

我们使用pytest进行测试。请确保您的更改包含相应的测试用例。

运行测试：
```bash
pytest
```

运行带覆盖率的测试：
```bash
pytest --cov=.
```

## Pull Request 流程

1. 确保您的分支基于最新的main分支
2. 确保所有测试通过
3. 更新相关文档
4. 提交清晰的提交信息
5. 等待代码审查

## 行为准则

请遵守以下行为准则：

- 尊重他人，保持友好和专业的交流
- 欢迎各种背景和经验水平的贡献者
- 建设性地提出批评和建议
- 专注于问题本身，而不是个人

## 联系方式

如有问题，可以通过以下方式联系：

- 创建GitHub Issue
- 通过邮件联系维护者

感谢您的贡献！🎉
